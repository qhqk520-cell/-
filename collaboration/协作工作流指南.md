# 协作工作流指南

## 🎯 目标
建立高效、透明、实时的ds-mini协作工作流，最大化生产力。

## 🔄 工作流概览
```
日常交流 → ds_mini_chat.md (实时)
     ↓
任务讨论 → GitHub Issues (正式)
     ↓
进度跟踪 → ds_progress.md + status.json
     ↓
成果交付 → 相关文档文件
     ↓
归档总结 → 每日/每周报告
```

## 📱 实时交流层 (ds_mini_chat.md)
### 使用场景
- 快速问题确认
- 日常进展通报
- 简单协调沟通
- 紧急事项通知

### 规则
1. **格式**: `[YYYY-MM-DD HH:MM] [发送者]: 内容`
2. **响应时间**: 2小时内回复
3. **归档频率**: 每天结束时归档到 `chat_archive/`
4. **内容要求**: 简洁明了，避免长篇大论

## 🎫 正式讨论层 (GitHub Issues)
### 使用场景
- 任务详细讨论
- 问题深入分析
- 决策记录
- 里程碑评审

### 规则
1. **每个任务一个Issue**
2. **使用模板创建**
3. **标签分类管理**
4. **关闭表示完成**

## 📊 进度跟踪层
### ds_progress.md (ds专属)
- ds的个人工作记录
- 详细任务分解
- 成果清单
- 能力验证记录

### status.json (共享状态)
- 实时项目状态
- 双方进度
- 沟通记录
- 下一步计划

## 📁 文件管理
### 目录结构
```
collaboration/
├── ds_progress.md          # ds工作进度
├── ds_mini_chat.md         # 实时交流
├── status.json             # 共享状态
├── 助手对话记录.md         # 历史对话
├── github_issue_template.md # Issue模板
├── 协作工作流指南.md       # 本文件
└── chat_archive/           # 聊天归档
    └── YYYY-MM-DD.md
```

### 文件更新频率
- **实时文件**: 随时更新 (ds_mini_chat.md)
- **状态文件**: 重要事件时更新 (status.json)
- **进度文件**: 每日更新 (ds_progress.md)
- **对话记录**: 每次重要对话后更新

## ⏰ 时间安排
### 每日节奏
- **09:00**: 晨间同步 (通过ds_mini_chat.md)
- **12:00**: 午间进展更新
- **15:00**: 问题协调时间
- **18:00**: 当日总结
- **21:00**: 明日计划

### 每周节奏
- **周一**: 周计划制定
- **周三**: 中期评审
- **周五**: 周总结和下周计划

## 🚨 紧急处理
### 问题分级
1. **紧急** (1小时内响应)
   - GitHub推送失败
   - 网络完全中断
   - 关键文件损坏

2. **重要** (4小时内响应)
   - 任务遇到重大障碍
   - 需要对方紧急协助
   - 进度严重滞后

3. **一般** (24小时内响应)
   - 常规问题咨询
   - 进度更新
   - 文件评审请求

### 升级路径
```
ds_mini_chat.md → GitHub Issue → 墨笔大人中转
```

## 📈 效果评估
### 关键指标
1. **响应时间**: 平均回复时间
2. **问题解决率**: 问题解决比例
3. **进度达成率**: 里程碑完成情况
4. **沟通效率**: 信息传递准确性

### 定期评审
- **每日**: 快速回顾当日沟通效果
- **每周**: 系统评估工作流效率
- **每月**: 优化调整工作流规则
