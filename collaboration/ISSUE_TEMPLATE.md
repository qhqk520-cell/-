# GitHub Issue 协作模板

## 使用说明
此模板用于ds和mini之间的任务讨论和实时沟通。每个重要任务或讨论主题应创建一个独立的Issue。

## Issue 创建格式

### 标题格式
`[任务类型] 任务名称 - 创建者 - 日期`

示例：
- `[T001] 文献资料库建设讨论 - mini - 2026-03-24`
- `[T002] 研究框架设计协作 - ds - 2026-03-24`
- `[讨论] 实时沟通方式探讨 - ds - 2026-03-24`

### 标签系统
- `task-T001` - 文献资料库相关
- `task-T002` - 研究框架相关
- `discussion` - 一般讨论
- `urgent` - 紧急事项
- `help-needed` - 需要帮助
- `tool-feedback` - 工具反馈
- `decision` - 需要决策

## Issue 内容模板

- **预计解决时间**: [YYYY-MM-DD]

[详细描述需要讨论的问题或需要完成的任务]

## 当前状态

## 评论格式要求

### 基本格式
```
**[来自]**: [ds/mini]
**[时间]**: [YYYY-MM-DD HH:MM]
**[类型]**: [回复/提问/建议/决策/更新]

**内容**:
[你的评论内容]

**引用**:
> [如果需要引用之前的评论]

**附件**:
- [文件链接或描述]
```

### 评论类型说明
- **回复**: 回答对方的问题
- **提问**: 提出新的问题
- **建议**: 提出建议或方案
- **决策**: 做出决定或确认
- **更新**: 提供进展更新
- **紧急**: 需要立即关注的紧急事项

## 协作规则

### 1. 响应时间
- **普通问题**: 在下一个60分钟更新周期内回复
- **紧急问题**: 使用`@mention`并标记`urgent`，期望2小时内回复
- **决策事项**: 24小时内达成共识或明确下一步

### 2. 通知机制
- 使用`@ds`或`@mini`直接提及对方
- 重要更新使用GitHub通知功能
- 紧急事项同时在聊天文件中标记

### 3. 问题解决流程
```
发现问题 → 创建Issue → 讨论分析 → 提出方案 → 实施解决 → 关闭Issue
```

### 4. 记录要求
- 所有重要决策必须在Issue中记录
- 实施结果需要更新到相关文件
- 关闭Issue时需要总结解决方案

## 示例 Issue

### 标题
`[T001] 文献分类系统优化讨论 - mini - 2026-03-24`

### 内容
```markdown

- **创建者**: mini
- **创建时间**: 2026-03-24 15:20
- **关联任务**: T001
- **优先级**: 中
- **预计解决时间**: 2026-03-25

## 问题/任务描述
当前文献分类系统需要优化，特别是大乘经典下的子分类不够细致。

## 当前状态
已建立4个一级分类，但二级分类需要细化。

## 需要讨论/决策的事项
1. 大乘经典是否应该按学派进一步细分？
2. 是否需要添加"研究热点"作为独立分类维度？

## 建议方案
1. 在大乘经典下添加学派维度：中观派、瑜伽行派、如来藏系
2. 保留"研究热点"作为标记而非独立分类
3. 建立三级分类系统：一级(4类) → 二级(学派/类型) → 三级(具体经典)

## 需要对方提供的帮助
1. ds能否从技术角度评估三级分类的可行性？
2. 是否需要为分类系统创建专门的数据库表？

## 下一步行动
- [ ] ds提供技术可行性分析
- [ ] mini细化具体分类方案
- [ ] 共同确定最终分类结构

## 相关资源
- `collaboration/共享文献数据库设计.md`
- `collaboration/ds_mini_chat.md#分类讨论`
```

## 紧急沟通流程

### 情况1：技术问题阻碍工作
1. 创建Issue标记`urgent`和`help-needed`
2. 在聊天文件中添加紧急通知
3. 使用`@mention`确保对方看到
4. 描述问题现象和影响范围

### 情况2：需要立即决策
1. 创建Issue标记`urgent`和`decision`
2. 明确决策选项和截止时间
3. 提供必要的背景信息
4. 请求对方在指定时间内回复

### 情况3：协作中断
1. 检查对方最后活跃时间
2. 如果在预期时间内无响应，创建Issue
3. 同时在聊天文件中记录
4. 通知墨笔大人（如果需要）

## 最佳实践

### 保持Issue整洁
- 一个Issue一个主题
- 定期总结讨论进展
- 无关评论移到新Issue
- 已解决的问题及时关闭

### 有效沟通
- 明确问题，提供背景
- 提出具体、可操作的建议
- 尊重对方专业领域
- 保持积极协作态度

### 文档同步
- 重要决策更新到相关文件
- 技术方案记录到设计文档
- 进度更新反映到状态文件
- 经验教训记录到MEMORY.md
