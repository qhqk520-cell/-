# 共享文献数据库设计草案

## 🎯 设计目标
基于mini的文献整理工作和ds的系统设计能力，创建一个高效、可扩展的共享文献数据库，支持佛学研究协作。

## 📊 数据库架构

### 1. 核心表设计
#### 1.1 文献主表 (literature_main)
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | INT PRIMARY KEY | 唯一标识 | 1 |
| title_zh | VARCHAR(255) | 中文名称 | 《长阿含经》 |

| author | VARCHAR(255) | 作者/编者 | 佛陀弟子结集 |
| compilation_period | VARCHAR(100) | 编撰时期 | 公元前5-4世纪 |
| original_language | VARCHAR(50) | 原始语言 | 梵文/巴利文 |
| translation_versions | JSON | 翻译版本 | {"中文": ["鸠摩罗什译", "玄奘译"], "英文": ["BDK English Tripitaka"]} |
| category_id | INT | 分类ID | 1 |
| importance_level | INT | 重要性等级 | 5 |
| research_hotspot | BOOLEAN | 研究热点标记 | true |
| digital_status | VARCHAR(50) | 数字化状态 | 已数字化 |

| summary_en | TEXT | 英文摘要 | Fundamental texts of Early Buddhism... |
| created_at | TIMESTAMP | 创建时间 | 2026-03-24 13:40:00 |
| updated_at | TIMESTAMP | 更新时间 | 2026-03-24 13:40:00 |

### 3. 重要性评级标准
#### 3.1 文献重要性 (1-10分)
- 10分：根本经典，研究必需
- 8-9分：重要经典，深度研究需要
- 6-7分：一般经典，参考价值
- 4-5分：次要文献，补充参考
- 1-3分：边缘文献，偶尔参考

#### 3.2 研究价值评估维度
1. **学术价值**：对学术研究的贡献度
2. **研究潜力**：未来研究可能性
3. **可获取性**：获取和使用的便利性
4. **完整性**：文本保存完整程度

### 4. 数据库功能设计
#### 4.1 核心功能
1. **文献检索**：按名称、分类、概念、时期检索
2. **概念关联**：查看概念相关的所有文献
3. **价值评估**：基于多维度评估文献价值
4. **进度跟踪**：跟踪数字化和整理进度
5. **协作标注**：支持多人协作标注和评论

#### 4.2 自动化功能
1. **批量导入**：支持Excel/CSV批量导入
2. **数据验证**：自动检查数据完整性和一致性
3. **进度计算**：自动计算整理进度百分比
4. **报告生成**：自动生成统计报告

#### 4.3 协作功能
1. **版本控制**：文献信息的版本管理
2. **修改记录**：记录所有修改历史
3. **讨论区**：每篇文献的讨论空间
4. **任务分配**：文献整理任务分配和跟踪

### 5. 技术实现方案
#### 5.1 数据库选择
- **主数据库**：SQLite（轻量级，适合协作）
- **备份方案**：Git版本控制 + 定期导出

#### 5.2 前端界面
- **Web界面**：简单易用的Web管理界面
- **搜索功能**：全文搜索 + 高级筛选
- **数据可视化**：进度图表、分类分布图

#### 5.3 自动化工具
- **Python脚本**：数据导入、验证、导出
- **Git集成**：自动提交数据库变更
